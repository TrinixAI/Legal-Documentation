# AccessScan Privacy Policy

**Effective Date:** May 16, 2026  
**Provider:** Srdjan Lopez LLC dba Trinix AI  
**Product:** AccessScan

This Privacy Policy explains how Srdjan Lopez LLC dba Trinix AI collects, uses, shares, and protects information in connection with AccessScan.

AccessScan is an AI-assisted accessibility compliance workflow platform. It helps customers identify potential accessibility issues, manage remediation, generate reports, and receive guidance related to Section 508, WCAG, and digital accessibility practices.

## 1. Important Compliance Notice

AccessScan assists with accessibility compliance workflows. It does not certify legal compliance.

AccessScan does not guarantee compliance with WCAG, ADA, Section 508, EN 301 549, or any other accessibility law, regulation, or standard. Automated scan results, AI guidance, document checks, reports, analytics, and remediation suggestions require human review.

## 2. Information We Collect

### Account and Contact Information

We may collect names, work email addresses, organization names, job titles, billing contacts, support contacts, authentication identifiers, workspace memberships, and role information.

### Customer Configuration Data

We may collect workspace settings, project settings, scan settings, crawl limits, platform profiles, role and permission settings, notification settings, AI provider settings, CI/CD settings, and API key metadata.

API key metadata may include key name, creation time, last used time, permission scope, and revocation status. Full API key secrets should not be displayed after creation.

### Submitted URLs and Scan Targets

We may collect submitted URLs, domains, subdomains, crawl paths, page titles, HTTP status codes, redirect information, sitemap references, robots and crawl metadata, authentication flow metadata if configured, and platform context.

### Scan Results and Accessibility Findings

AccessScan may collect and generate potential accessibility issues, rule identifiers, WCAG or Section 508 references, severity levels, affected selectors, DOM paths, affected HTML snippets, ARIA attributes, color contrast data, screenshots if enabled, scan timestamps, scan status, remediation status, comments, assignments, and exported report data.

### Uploaded Documents

When document checks are used, we may collect and process PDF, DOCX, and PPTX files, file names, file size, file type, page count, slide count, document title, language, parser used, extracted text or snippets, document structure signals, reading order indicators, table structure indicators, alt text indicators, issue details, status history, and generated reports.

### AI Usage Data

If AI features are enabled, we may collect prompts, user requests, issue context, scan findings, selected HTML snippets, selected document excerpts, AI responses, model or provider metadata, request timestamps, token usage, latency, error metadata, feature settings, and approval or edit history where available.

### Audit Logs and Security Data

We may collect login events, user actions, role changes, API key events, scan events, report events, integration activity, IP address, device and browser information, error logs, security alerts, and abuse prevention signals.

### Support and Communications

We may collect support tickets, emails, demo requests, pilot intake forms, meeting notes, customer feedback, product requests, bug reports, billing communications, and contract communications.

- Manage users, roles, permissions, workspaces, and API keys
- Support CI/CD and API workflows

## 4. AI Processing

AccessScan may use AI to explain accessibility issues, generate remediation guidance, summarize findings, draft action plans, classify issues, and answer Section 508 or WCAG questions.

Depending on configuration, AI processing may involve Trinix AI-managed providers, third-party AI providers, customer-selected providers, locally hosted models, or other supported configurations.

When needed to provide an AI feature, limited relevant context may be sent to the configured AI system. This may include scan findings, affected HTML snippets, selected document excerpts, issue metadata, project context, and user prompts.

We do not use customer content to train public AI models unless expressly stated in a separate written agreement or customer-enabled setting.

Customers are responsible for confirming that any enabled AI provider is approved for the customer's data classification and environment.

## 5. How We Share Information
