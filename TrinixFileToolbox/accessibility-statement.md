# Accessibility Statement

**Trinix File Toolbox**
A desktop product of Srdjan Lopez LLC dba Trinix AI
Effective Date: 2026-05-20
Version: 1.0

---

## 1. Our Commitment

Trinix File Toolbox is designed to be usable by as many people as practical,
including users who rely on keyboard navigation, screen magnification, high
contrast settings, or other assistive configurations. Accessibility is
treated as part of product quality, not an add-on.

## 2. Conformance Target

We target the principles of the Web Content Accessibility Guidelines
(WCAG) 2.1 Level AA where they apply to desktop application interfaces.
WCAG was authored for the web, so some criteria do not map cleanly to a
desktop GUI; we apply the spirit of each criterion where the desktop
context permits.

## 3. Features Currently Supported

- **Keyboard navigation.** All primary actions are reachable by Tab,
  Shift+Tab, arrow keys, Enter, and Space. Sidebar pages, segmented
  controls, buttons, links, and inputs participate in the focus order.
- **Visible focus indicators.** Focused controls show a cyan focus ring.
  Active sidebar items show a left accent and a stable border that does
  not cause text clipping.
- **Scalable text.** A font scale control in Settings provides Small,
  Default, Large, and Extra Large options. The sidebar and content panes
  resize to accommodate the chosen scale.
- **High contrast styling.** A dark theme is used by default with strong
  text-on-background contrast for primary content.
- **Accessible names.** Interactive controls are given accessible names
  intended for screen readers where the Qt accessibility bridge is
  available.
- **No reliance on color alone.** State is conveyed by shape, label, and
  layout in addition to color.
- **Local-only operation.** No required cloud step, account, or sign-in
  flow blocks access to the Software.

## 4. Known Limitations

- Screen-reader coverage on Windows depends on the Qt accessibility bridge
  and on the specific screen reader in use. Coverage for complex custom
  widgets may be partial.
- Some auto-generated content such as PDF text extraction reflects the
  structure of the source document and may not be perfectly ordered for
  assistive technology.
- The Software targets desktop Windows and is not designed for touch-first
  or mobile use.

## 5. Feedback

If you encounter an accessibility barrier in Trinix File Toolbox, please
contact <support@trinix.gg> with:

- A short description of the issue.
- The page or feature where it occurred.
- Your operating system version and any assistive technology you were
  using (for example, "Windows 11 with Narrator").

We treat accessibility reports as priority bugs.

## 6. Updates

This statement will be reviewed when the user interface changes in a way
that affects accessibility, and at minimum with each significant release.
